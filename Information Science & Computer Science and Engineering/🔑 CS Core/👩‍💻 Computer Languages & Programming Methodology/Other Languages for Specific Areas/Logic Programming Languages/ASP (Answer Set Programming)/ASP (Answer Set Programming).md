## Intro
> 🔗 https://en.wikipedia.org/wiki/Answer_set_programming

**Answer set programming** (**ASP**) is a form of [declarative programming](https://en.wikipedia.org/wiki/Declarative_programming "Declarative programming") oriented towards difficult (primarily [NP-hard](https://en.wikipedia.org/wiki/NP-hard "NP-hard")) [search problems](https://en.wikipedia.org/wiki/Search_algorithm "Search algorithm"). It is based on the [stable model](https://en.wikipedia.org/wiki/Stable_model_semantics "Stable model semantics") (answer set) semantics of [logic programming](https://en.wikipedia.org/wiki/Logic_programming "Logic programming"). In ASP, search problems are reduced to computing stable models, and _answer set solvers_—programs for generating stable models—are used to perform search. The computational process employed in the design of many answer set solvers is an enhancement of the [DPLL algorithm](https://en.wikipedia.org/wiki/DPLL_algorithm "DPLL algorithm") and, in principle, it always terminates (unlike [Prolog](https://en.wikipedia.org/wiki/Prolog "Prolog") query evaluation, which may lead to an [infinite loop](https://en.wikipedia.org/wiki/Infinite_loop "Infinite loop")).

In a more general sense, ASP includes all applications of answer sets to [knowledge representation and reasoning](https://en.wikipedia.org/wiki/Knowledge_representation_and_reasoning "Knowledge representation and reasoning") and the use of Prolog-style query evaluation for solving problems arising in these applications.
