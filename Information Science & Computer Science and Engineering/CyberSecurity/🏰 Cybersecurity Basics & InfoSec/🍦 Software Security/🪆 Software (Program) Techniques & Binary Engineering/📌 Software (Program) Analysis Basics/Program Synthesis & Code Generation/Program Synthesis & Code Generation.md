## Intro
> 🔗 https://en.wikipedia.org/wiki/Program_synthesis

In [computer science](https://en.wikipedia.org/wiki/Computer_science "Computer science"), **program synthesis** is the task to construct a [program](https://en.wikipedia.org/wiki/Computer_program "Computer program") that [provably](https://en.wikipedia.org/wiki/Provably_correct "Provably correct") satisfies a given high-level [formal specification](https://en.wikipedia.org/wiki/Formal_specification "Formal specification"). In contrast to [program verification](https://en.wikipedia.org/wiki/Program_verification "Program verification"), the program is to be constructed rather than given; however, both fields make use of [formal proof](https://en.wikipedia.org/wiki/Formal_proof "Formal proof") techniques, and both comprise approaches of different degrees of automation. In contrast to [automatic programming](https://en.wikipedia.org/wiki/Automatic_programming "Automatic programming") techniques, specifications in program synthesis are usually non-[algorithmic](https://en.wikipedia.org/wiki/Algorithm "Algorithm") statements in an appropriate [logical calculus](https://en.wikipedia.org/wiki/Formal_system "Formal system").

The primary application of program synthesis is to relieve the programmer of the burden of writing correct, efficient code that satisfies a specification. However, program synthesis also has applications to [superoptimization](https://en.wikipedia.org/wiki/Superoptimization "Superoptimization") and inference of [loop invariants](https://en.wikipedia.org/wiki/Loop_invariant "Loop invariant").
