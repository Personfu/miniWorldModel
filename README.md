## Reading Guidance
The note is entirely written under obsidian. Hence, it is best rendered under obsidian as you would see it exactly the way I see it (FYI I'm using the theme 🔗 [atom](https://forum.obsidian.md/t/atom-theme/1238) ). If you are browsing this from github webpage, some contents would look ugly and maybe even un-readable. Anyway, I suggest to clone the whole repo to local and read it from obsidian.

An important plugin you should have on obsidian is 🔗 [excalidraw](https://obsidian.md/plugins?search=excalidraw). Since there are lots of diagrams I draw based on this. If you don't have it you cannot see those diagrams, and that would be a big miss.
- Inside my diagram in excalidraw, there are links as well. This way, you can jump from one diagram directly to another diagram or file.

Another tip is, this note is a personal knowledge base after all. Hence, there are some files that I choose not to share on the public, so maybe this would cause some difficulty when others try to read under related contexts. I cannot help with this issue. Sorry. However, unless it is about personal privacy or copy rights, usually there would be a link attached to the resources originally meant to be displayed. In that case, you can still access to the resources I want to share by the link.

Also, the notes was created ever since my first semester in the university. This means, there might be some early notes that I rarely revisit since I created them. For those early notes, they are at a lower quality.

Last tip (I suggest you to do so since this is what I am doing), always read structure first, before dive into the depth of contents. This means, if you opened some file, have a look at its folder structure first. For example, have a look at the name of its parent folder, brother folders, grandpa folder, etc. Also, when reading a note file, read the outline of the file first, and use that to navigate around.


### Structure of Single Note File
For those notes in a mature quality, they follow the following template in general: 

```md
# Title

## Res
### Related Topics

### Other Resources


## Intro

## Ref
```

Where `Res` lists related resources (other notes or webpages, books, etc.)  to the contents of this file, while `Ref` lists the references in the file. `Intro` is reserved for the body of the file. Usually, between `Res` and `Ref` there would be much more than just an `Intro`.

Specifically, under `Res`, `Related Topics` lists other notes in this notebook.  For the books, videos, websites, etc, they are in the rest sub-sections of `Res` (for example, `Other Resources`).


### Searching Among Files
The whole notebook is large. Although it is well-organized as a mind map, with all the internal links, it could still be hard to quickly navigate through them. This is why we need to search.

During my writing, I mainly use three search methods:
- file search: in obsidian, `ctl-o` opens the file search bar.
- text search: in obsidian, I use a plugin 🔗 [OmniSearch](https://obsidian.md/plugins?search=Omnisearch). This allows user to search a text in all files.
- tag search: in my notes sometimes I will use tags for contents that's a FAQ, or those paragraphs /sub-sections where I compares two concepts. So tag search might not give you everything you want to search.

There are many other search options supported by obsidian. You are welcomed to give them a try.
